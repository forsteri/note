話題になったもののざっくり雑感。2026年3月における個人的な観測。
普段追いかけているニュースや記事から感じた見立てを並べたもの。

## Cursor

Cursor。IDE統合型の走り。Claude vs GPTのモデル層とは別に、IDEとの統合やエージェントモードによる性能と使い心地が一世を風靡した。当初は確実にVSCode + GitHub Copilotの先を行っていた。しかし、料金体系の改定やKiro, AntigravityなどのIDEの登場やClaude Code, Gemini CLI, CodexといったCLI型の発展により、今は話題性では下火。IDEが乱立しても面倒で、エディタやIDEは好きなもので、それに加えてCLIを使うスタイルが好まれているように見える。新興企業なので、大企業にとっては採用と契約にハードル。Cursorを使っている層はイノベーター寄りで、今後別のIDEやCLIが優位と見ると、あまり躊躇わずに移行しそうにも思える。

現時点の難点は、トークンベースの課金らしい。コストパフォーマンスが落ちた？

## VSCode + GitHub Copilot

言わずと知れたIDE・エディタのデファクトスタンダード。IDEに関して、AI以前は個人の好みと特に制約がなければこれ一択だった（オープンな開発界隈では）。後続もこれをフォークしているあたり、エディタとしての完成度は高いのだろう。しかしながら、Microsoft自身の開発は難航しているのか、AI統合は後手後手でCursorの後塵を拝していた。後追いかつ物足りなかった印象。だが、Microsoftの看板効果、持久戦をすれば最後にはある程度のポジションに収まっているのではないかという見込みと、契約のハードルの低さから使っている企業は多そう。確実に開発が進むのには安心感がある。個人開発者も、GitHub Copilotは安価で使いやすい。自分でちょっとした編集をするときの補完は地味に便利。コストパフォーマンスの高さから今も有力な選択肢になっている。現時点で、月10ドルでPremiumリクエスト300回。

## Microsoft Copilot

単独での評価は難しい。GPTが搭載されるまでは勝負にすらなっていなかったし、GPTが搭載されても内部的な調整のためか純正のGPTやClaudeの方が良かった印象。しかし、Office統合とエンタープライズ保護、すなわち、業務用のPCで気兼ねなく使えるのが強すぎて、ユーザーへのリーチと単純なチャット相手としては最も多いかもしれない。Edgeとの統合で、気軽なチャットと調べものの使い勝手も向上した。コーディングエージェントとは別ジャンル。私見。Microsoftのカンファレンスに参加したときのこと。ExcelやPowerPointの資料が良い感じにサクッとできる、というデモはITに明るくなく、それらの資料を見たり作ったりする経営層・意思決定層に非常に有効に働くのだろうと思った。Microsoftは、Windows, Office, VSCodeというUI・エンドポイントを押さえているのがあまりに強い。そして上位層を押さえるマーケティング・営業力も強い。カンファレンスは、ギーク向けのAWS、エグゼクティブ向けのMSの印象を強く感じた。なお、その割に開発はそれほど強くない。

## GeminiとAntigravity

まずはGemini。技術力に評価があるGoogleだが、モデル競争の序盤、Bardでボコボコになったところからよくここまで盛り返した。技術力の土台と資金力を感じる。ほとんど使ったことがないので実体験では判断できないが、評判を聞いている限り、コーディング能力の評価は高い。半面、ハルシネーションの問題も聞いた。ただ、評判だけから推測するに、現在のモデル性能は拮抗してきていると感じている。その中で、十分に強いモデルなのだろう。100万トークンのコンテキストウィンドウは他の主要モデルと比べても強力。Windows + Officeとは違ったGoogleのエコシステムとの統合が強い。モデル（画像や動画生成も）、コーディングエージェント向けのCLI、プラットフォーム・アプリ（Google Workspace, Notebook LM）、自前でフルラインナップを取り揃えている。Google Workspaceを使っていれば自然と選択肢になりそう。

それに加えてAntigravity。まだ全然使っていないので評価はできない。新しい物好きを除いて、実際こういう人は多いのではないだろうか。IDEを出されても切り替えるのが面倒で、そろそろ食傷気味。それを覆すほどの魅力はないが、Googleで揃えるのなら良いのかもしれない。

追記。3/12にサービスの改定で失望が広がっている。
- 利用上限の引き下げ
- 課金プランのリセットの短縮
- Gemini 3.1 Pro, Claude Sonnet/Opusといった上位モデルがほぼ使えなくなる
上位モデルを使うには、月額36,400円のUltraプランが必要らしい。

## Kiro

AWSがリリースしたVSCodeフォークのIDE。仕様駆動開発（SDD）のコンセプトを提示したところに新規性がある。が、総合的なAI統合IDEとしての実力は既存の勢力図を切り崩せるほどでもなく、実際のシェアは限定的な印象。これ単体では仕様の変更（実装した後に改善するというサイクル）に弱いとも聞く。コンセプトは受け入れられて、SDDを志向するツールが作られたり、他のツールに取り込まれたりした。発想は良かったが、ツールとして広まりきらず、他に取り込まれる。ITではよくあるパターン。ちなみに、AWSの中の人（1,500人！）からKiroではなくClaude Codeを使いたいとの申請があったとか。
私見。Q Developerの再編など、サービス形態とブランディングが揺れていてややこしい。

## Devin

リストアップしたものの、今の話題は非常に限定的。Git(GitHub)駆動のリモートエージェントという点で画期的だった。使ったことはない。コーディング特化な上、個人向けには高価で使いづらかった。GitHubに常駐して、Issueを起点にまるでリモート勤務のメンバーのように動いてくれる点がウケていた。使われるところではまだ使われているのだろうが、話題になるほどではなくなった。確かここの会社がWindsurfを買っていた。
補足。非同期で動いてくれることは実はそれほど嬉しくなかったのかもしれない。つまり、コンセプトの目新しさや先進的なスタイルがウケたけど、今のモデル・ツールの精度とスピードからすると対話的な利用で十分かつ小回りが利く。ただ、使われているところでは使われているのだろう。個人での使い勝手は良くないので、あまり話題には上がりにくい。組織が抱える課題によるが、常駐して半自律で開発を進めてくれるAIはまだ今の現場にフィットしていないのかもしれない。開発自体よりその前後にフォーカスが移動している印象を受けている。なお、自律度合いとこなせる課題のサイズが大きくなればまた出番が増えるかもしれない。

## GitLab Agent Platform

実はこれについては機能を把握できていない。GitHubとCopilotに相当する機能を持っているのか？Gitの基盤がGitLabな組織も多いはずで、そこでは選択肢に上がるはず。話題性がなさすぎて、GitLabとの親和性や機能統合の観点から選ぶものであって、GitHubから移行するほどのものではないのではないか。逆に、GitLab + GitHub Copilotはあまりなさそう。GitLabは、エンタープライズのユーザーが多く、話題が上がりにくいのだろう。それにしても特に優れているとは聞かない。GitHub Copilotにしてもそうだが、Claude CodeやGemini CLIのようなツールとの差別化が難しい。結局、ローカルで稼働して、コードベースを読み取り、出力し、GitLab/GitHubのCLIやAPIを使えればそれで済むことがほとんど。Devinのような常駐型としての動きが期待されるか。しかし改めて考えると、リモートリポジトリを更新する権限を持っているのはけっこう危ういのではないだろうか。プラットフォームに統合された機能に固執する必要性はない認識。
追記。コーディング支援というより、エージェントオーケストレーションに振っていて、Planner Agent（バックログ→タスク分解）、Security Analyst Agent（脆弱性分析・トリアージ）、Data Analyst Agent（メトリクス分析）といった役割特化のエージェントが並列で動く…らしい。複数エージェントを連携してワークフローを自動化できるとか。コードとDevSecOpsの基盤という感じ。

## ChatGPT/Codex

これぞAI(LLM)ブームの開祖。モデルの性能でトップランナーだったが、GPT-5が高まりすぎた期待を超えられず、その後は目立っていない印象。「性格」が問題になり4oが復活したりと、コーディング能力以外の面で話題になったことも。多様な用途で最も使われているモデルであることは間違いない。なお、目立っていない一因は、もはや多くの人々がモデルのアップデートを評価できなくなっていることにも起因する。見方によっては既にAGI。先駆者であり、高性能なモデルのみを持っていて、その使い道と利益の出し方にずっと苦戦している。画像・動画生成をリリースしたり、ハードウェアとの統合を模索したり。Claude Codeに出遅れて話題を奪われたが、Codexはコーディング性能、特に設計やレビュー能力は高いらしい。加えてコストパフォーマンス。遅いが、賢く、大きなタスクにも対応できる。性格は頑ななところがあるらしい。Claudeが実装、Codexが設計、の組み合わせはちょくちょく耳にする。

## Claude/Claude Code

開発者はOpenAI出身。方向性に迷うChatGPTに対して、明確に開発用途を志向して差別化、確実な収益源を得つつある。当初に話題になっていたのは、性格の良さ・文章力・倫理。その後、ArtifactやClaude Codeと開発向けの機能が拡充されていった。今でこそ非開発者含めた大きなブームになっているが、Claude Codeはリリース当初は今ほど話題になっていなかった。その後の、特にフラグシップの推論モデルのOpusとSonnetの進展・組み合わせの強力さによるところが大きい。後になって脚光を浴びた形。今に至って、CoworkでSaaS不要論を巻き起こしたり、Claude Code Securityでセキュリティ株を暴落させたり、レガシーマイグレーションでIBM株を暴落させたり、一躍AI界の中心に躍り出た。何と言ってもエンタープライズ向けに堅い。モデルのコーディング能力が高く、開発向けの機能が充実しており、AWSのBedrockでプライベートに（非学習・データ隔離）使うことができる。GPTとの比較では、性格が悪いがめちゃくちゃ賢いGPTと性格の良い優等生なClaudeと対比される。なお、さっきも述べたが、もはやほとんど違いが判らない。Opus 4.5と4.6もほとんどわからない。確か2025年末くらいからClaude Codeの話ばかりになった印象。
